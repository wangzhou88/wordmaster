# 如何手动触发GitHub Actions工作流

## 步骤说明

1. **访问GitHub仓库**
   - 在浏览器中打开您的GitHub仓库链接

2. **进入Actions页面**
   - 点击仓库顶部的"Actions"标签页

3. **选择工作流**
   - 在左侧工作流列表中，找到并点击"📱 WordMaster APK构建"

4. **手动触发工作流**
   - 点击右侧的"Run workflow"按钮
   - 在下拉菜单中选择构建类型：
     - `debug`: 构建调试版本APK（推荐）
     - `release`: 构建发布版本APK
   - 点击绿色的"Run workflow"按钮

5. **监控工作流**
   - 工作流将开始运行，您可以点击工作流运行查看进度
   - 点击每个步骤可以查看详细日志

6. **等待构建完成**
   - 构建过程通常需要几分钟到十几分钟，请耐心等待
   - 构建完成后，您可以查看结果摘要

## 预期结果

工作流成功完成后，您将看到：
- 构建摘要包含APK文件信息
- "Artifacts"区域包含可下载的APK文件
- 详细的构建日志

## 下一步

构建完成后，请按照以下步骤下载APK文件：
1. 在工作流运行页面底部找到"Artifacts"区域
2. 点击"wordmaster-apk-{commit-hash}-{build-type}"下载压缩文件
3. 解压下载的文件，您将获得APK文件

## 故障排除

如果工作流失败，您可以：
1. 点击失败的工作流运行
2. 查看失败的步骤和详细日志
3. 根据错误信息调整代码或配置
4. 重新触发工作流
