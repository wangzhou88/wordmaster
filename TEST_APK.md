# 下载和测试生成的APK文件

## 下载APK文件

当GitHub Actions工作流成功完成后，您可以按照以下步骤下载生成的APK文件：

1. **查看工作流运行结果**
   - 在GitHub仓库的"Actions"页面中，找到已完成的工作流运行
   - 点击工作流运行进入详细页面

2. **下载APK文件**
   - 在工作流运行页面的底部，找到"Artifacts"区域
   - 点击"wordmaster-apk-{commit-hash}-{build-type}"下载压缩文件
   - 解压下载的文件，您将获得APK文件

3. **APK文件位置**
   - APK文件通常命名为`wordmaster-1.0.0-{architecture}.apk`
   - 例如：`wordmaster-1.0.0-arm64-v8a.apk`

## 在Android设备上测试APK

### 准备工作

1. **启用开发者选项**
   - 进入Android设备的"设置"
   - 滚动到底部，点击"关于手机"
   - 连续点击"版本号"7次
   - 返回"设置"，现在应该能看到"开发者选项"

2. **启用USB调试**
   - 进入"开发者选项"
   - 启用"USB调试"

3. **允许安装未知来源应用**
   - 进入"设置" > "安全"或"应用和通知"
   - 启用"未知来源"或"安装未知应用"
   - 选择您的文件管理器或浏览器，并允许安装

### 安装APK

1. **使用ADB安装**
   - 将APK文件复制到计算机
   - 连接Android设备到计算机
   - 打开命令行，输入：`adb install 路径/到/wordmaster-1.0.0-{架构}.apk`

2. **直接安装APK**
   - 将APK文件传输到Android设备
   - 在设备上找到APK文件
   - 点击APK文件开始安装
   - 按照提示完成安装

### 测试应用

1. **启动应用**
   - 在应用列表中找到"WordMaster"
   - 点击图标启动应用

2. **基本功能测试**
   - 检查应用是否正常启动
   - 测试主要功能是否正常工作
   - 注意任何错误或崩溃

## 故障排除

### 安装失败

1. **架构不匹配**
   - 确保APK架构与设备匹配
   - ARM设备使用`arm64-v8a`或`armeabi-v7a`版本
   - x86设备使用`x86_64`版本

2. **应用签名**
   - 如果是调试版本，可能需要卸载已存在的签名版本
   - 使用命令：`adb uninstall com.wordmaster.app`

### 应用崩溃

1. **查看日志**
   - 使用ADB命令查看日志：`adb logcat | grep WordMaster`
   - 在Android设备上查看日志：设置 > 开发者选项 > 显示崩溃日志

2. **常见问题**
   - 权限问题：检查应用权限设置
   - 资源文件缺失：确保所有必要的资源文件已正确打包
   - 依赖库不兼容：可能需要调整依赖项

## 总结

通过GitHub Actions构建APK是一种可靠的方法，特别是在Windows环境下无法直接构建APK的情况下。按照以上步骤，您可以下载并测试生成的APK文件，确保应用在Android设备上正常工作。
