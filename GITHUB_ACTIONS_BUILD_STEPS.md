# 🚀 GitHub Actions APK构建步骤指南

## 🎯 启动GitHub Actions构建

### 步骤1：访问GitHub Actions
1. **打开浏览器，访问：**
   ```
   https://github.com/wangzhou88/wordmaster/actions
   ```

2. **查看工作流列表**
   - 找到 "Build Android APK" 工作流
   - 点击进入工作流详情页

### 步骤2：运行工作流
1. **点击 "Run workflow" 按钮**（绿色按钮）
2. **选择分支：**
   - Branch: `main` （推荐）
   - 或选择您推送的分支
3. **点击绿色的 "Run workflow" 确认**

### 步骤3：等待构建开始
- 工作流将自动开始执行
- 您会看到黄色的"排队中"状态
- 然后变为蓝色的"运行中"状态

### 步骤4：监控构建进度
- **总耗时：** 约10-15分钟
- **主要阶段：**
  1. 检出代码 (1分钟)
  2. 设置Python环境 (2-3分钟)
  3. 安装依赖 (5-8分钟)
  4. 编译APK (3-5分钟)

## 📋 构建日志监控

### 如果遇到错误：
1. **点击失败的步骤**查看详细错误
2. **常见问题：**
   - 内存不足：降低并发构建数量
   - 依赖冲突：检查requirements.txt
   - 网络超时：重新运行工作流

### 成功标志：
- ✅ 所有步骤显示绿色勾号
- ✅ 生成APK文件
- ✅ 显示下载链接

## 📱 构建成功后

### 下载APK：
1. **滚动到页面底部**
2. **找到 "Artifacts" 部分**
3. **点击下载链接**
4. **保存APK文件到您的设备**

### 安装APK：
1. **将APK文件传输到Android设备**
2. **在设备上允许"未知来源"安装**
3. **点击APK文件安装**
4. **等待安装完成**
